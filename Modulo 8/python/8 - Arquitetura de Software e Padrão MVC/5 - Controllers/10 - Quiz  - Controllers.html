O que são controllers em uma aplicação web?

  • Classes responsáveis por gerenciar as requisições, validar a regra de negócio e retornar as respostas apropriadas.
  • Funções que definem o layout e a estilização de páginas web.
  • Componentes que lidam com a lógica de negócios e o acesso ao banco de dados.
  • Estruturas de dados usadas para armazenar informações de sessão do usuário.

Qual é o propósito dos testes unitários em desenvolvimento de software?

  • Testar a integração entre diferentes partes de um sistema.
  • Garantir que o software funcione conforme esperado do ponto de vista do usuário final.
  • Identificar e corrigir bugs no código.
  • Verificar se unidades individuais de código (como funções ou métodos) funcionam corretamente.

O que é um mock em testes de software?

  • - Uma simulação de um ambiente de produção para testar o desempenho do software. - B
  • Uma técnica para simular o comportamento de objetos ou componentes do sistema durante os testes.
  • Uma estratégia para evitar a execução de testes de forma a acelerar o desenvolvimento.
  • Um padrão de design usado para criar interfaces de usuário intuitivas e responsivas.

Qual é a principal diferença entre testes unitários e testes de integração?

  • Testes unitários verificam a integração entre diferentes partes do sistema, enquanto testes de integração verificam o comportamento de unidades individuais de código.
  • Testes unitários são automatizados, enquanto testes de integração são executados manualmente.
  • Testes unitários são rápidos de executar e isolam unidades de código específicas, enquanto testes de integração verificam a interação entre várias partes do sistema.
  • Testes unitários são usados apenas para verificar a lógica de negócios, enquanto testes de integração são usados para verificar a interface do usuário.

Por que usar mocks em testes unitários?

  • Para simular o comportamento real do código em produção.
  • Para isolar a unidade de código sendo testada e controlar o comportamento de suas dependências.
  • Para acelerar a execução dos testes, evitando a necessidade de acesso a recursos externos.
  • Para garantir que o código teste todas as possíveis ramificações de execução durante a execução do teste.
